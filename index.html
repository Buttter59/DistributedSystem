Features

Everything you need to demonstrate distributed systems

ChatIT covers all the requirements for a distributed systems demonstration — from network communication to multi-client handling.

Real-time Messaging

Messages broadcast instantly to all connected clients via a central TCP server using POSIX sockets and select()/threads.

Multi-client Support

The server spawns a thread per client, allowing many users to connect simultaneously without blocking.

Custom Username

Clients choose and change their display name at any time using the /name <username> command.

Private Messaging

Send direct messages to a specific user with /msg <username> <message> — visible only to the recipient.

Join/Leave Notifications

The server announces when any user connects or disconnects so everyone in the room is aware.

Stable Connection

Graceful error handling and client cleanup ensures the server stays alive even when clients crash or disconnect unexpectedly.

Architecture

How ChatIT works

A central server accepts TCP connections from many clients. Each client runs in its own thread on the server side for concurrent handling.

🖥️ Server responsibilities

  • → Listen on a TCP port for incoming connections
  • → Spawn a thread per connected client
  • → Receive messages and broadcast to all others
  • → Route private messages to specific clients
  • → Announce joins & departures

💻 Client responsibilities

  • → Connect to server via IP + port
  • → Send typed messages to the server
  • → Receive and display messages in real time
  • → Use commands: /name, /msg, /quit
  • → Run a background thread for incoming messages
